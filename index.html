Welcome! This is a walkthrough of the current setup for the server-client communication. It is to showcase the basis of creating a server-client communication between a nodejs server and C++ client.

Note: It is not an advanced guide and source code provided require JSON file to run! Please change accordingly!

Getting Started!

Before anything can be done, there are a few things you'll need to gather before the programming part. The most fundamental is Visual Studio - to create and run the client, and Node.js with an IDE of your choosing (this walkthrough will be using Atom) - to create and run the server. It will goes through the basic so skip to the step you're unfamiliar with.

To go straight to coding section, navigate through the navigation bar at the top!

Setting Up Node.js

  1. Download and install Node.js from https://nodejs.org/en/download/
  2. The installer will automatically configure your command prompt (adding to environment path, as shown below). If not, manually navigate to environment path by Search → "Edit the system environment variables" → "Environment Variables..." → "System Variables" → Double-Click "Path" → "New" → Add the directory of Node.js
  3. Test if Node.js is properly installed by running command prompt and typing "node -v". To run any javascript, navigate its directory and type "node filename.js" or "node filename"
  4. Done! Next is to setup the required library for server-client communication over here

Command Line Quick Guide:

  1. "cd pathname” change the current directory to target directory
  2. “cd..” backspace current directory
  3. "dir” view content of the current directory"

Setting Up Visual Studio

  1. Download Visual Studio 2017 Community Version from https://visualstudio.microsoft.com/thank-you-downloading-visual-studio/?sku=Community&rel=15
  2. Run the installer and under “Workloads”, select “Desktop development with C++” and click install
  3. After installation, launch visual studio. Navigate from “File” → “New” → “Project” to create a new C++ program project
  4. Under “Installed”, select “Empty Project”. If desired, change the “Name”, “Location” and “Solution name” and then clicked “OK”.
  5. Under “Solution Explorer”, right-click and navigate “Add” → “New Item…” and select “C++ File (.cpp)” and change the “Name” if desired.
  6. Done! You can start writing code. To setup libraries for server-client communication over here

WS Library - https://github.com/websockets/ws
  1. In command prompt, navigate to your working directory containing the javascript. Run “npm install --save ws” in the current working (where the js script reside) directory to install the library
  2. In the script, add const WebSocket = require(‘ws’); and done! You can now use the library. For usage, look at the documentation

Client-Server Communication Library - https://github.com/zaphoyd/websocketpp
  1. Download the library from the link as zip or using GIT software (download from https://git-scm.com/downloads)
  2. Unzip the file and place it in an accessible directory
  3. Right-Click in “Solution Explorer” and select “Properties” or Under “Project” → “Properties”
  4. Under “C/C++” → “General” → “Additional Include Directories” and add the library file (called websocketcpp) by navigating to “/../websocketcpp”
  5. Download the boost library from https://www.boost.org/users/download/
  6. Unzip the file to an accessible directory then find and run bootstrap.bat then b2.exe and bjam.exe.
  7. After completion, a file called “stage” will be generated. Go back to visual studio, “Project” → “Properties”, under “Linker” → “Additional Library Directories” and add the new file “/../stage/lib”.
  8. Under “C/C++” → “General” → “Additional Include Directories” and also add the library file (default called boost_X_XX_X) by navigating to “/../boost_X_XX_X”
  9. Done! Now the library can be used. Read up on this documentation about the usage as it's the foundation of the project
JSON Library - https://github.com/nlohmann/json
  1. Download the library from the link as zip or using GIT software (download from https://git-scm.com/downloads)
  2. Unzip the file and place it in an accessible directory
  3. Right-Click in “Solution Explorer” and select “Properties” or Under “Project” → “Properties”
  4. Under “C/C++” → “General” → “Additional Include Directories” and add the library file (called json) by navigating to “/../json/single_include”
  5. Done! Now the library can be used. Read up on this documentation about the usage as it's the foundation of the project
Encoding/Decoding Base64 Library - https://github.com/ReneNyffenegger/cpp-base64
  1. Download the library from the link as zip or using GIT software (download from https://git-scm.com/downloads)
  2. Unzip the file and place it in an accessible directory
  3. Right-Click in “Solution Explorer” and select “Properties” or Under “Project” → “Properties”
  4. Under “C/C++” → “General” → “Additional Include Directories” and add the library file (called cpp-base64) by navigating to “/../cpp-base64”. If it does not work, instruction here to add .h and .cpp file in “Solution Explorer”
  5. Done! Now the library can be used. Read up on this documentation about the usage as it's the foundation of the project
<base64.h> not working! What to do next?

The easiest solution is to manually add it like a normal header file!

  1. Add new item to your project OR copy and paste the base64.h and base64.cpp to your working directory (/projectname/projectname/) of your main.cpp
  2. Add one .cpp and one .h file to your project
  3. Open the download base64.h and base64.cpp file and copy and save in your project
  4. Use "base64.h" instead of <base64.h> and done! It should work.
